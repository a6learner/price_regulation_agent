# 三方法评测结果综合分析

**Last Updated**: 2026-04-18

**评测集**: `data/eval/eval_dataset_v4_final.jsonl`（780 条：违规 500，合规 280）

**评测口径**: RAG 与 Agent 均采用 `cases_k=0`（不注入案例检索，仅法规增强），防止同源污染。

---

## 1. 核心指标总览

| 指标 | Baseline | RAG | Agent |
|------|----------|-----|-------|
| **二分类准确率** | 89.86% | 89.87% | 88.33% |
| **违规类型准确率** | 70.09% | 70.64% | 69.25% |
| **Precision** | 90.25% | 89.05% | 87.39% |
| **Recall** | 94.40% | 96.00% | 95.60% |
| **F1** | 92.28% | 92.40% | 91.31% |
| 平均响应时间 | 5.85s | 6.00s | 25.6s |

**关键发现**：三种方法在分类层指标上差异很小（准确率均在 88%–90% 区间）。RAG 的 F1 最高（92.40%），主要得益于最高的 Recall（96.00%）。Agent 准确率略低，但 Recall 仍优于 Baseline。

---

## 2. 混淆矩阵对比

| | Baseline | RAG | Agent |
|--|----------|-----|-------|
| TP（正确识别违规） | 472 | 480 | 478 |
| TN（正确识别合规） | 228 | 221 | 211 |
| FP（误报） | 51 | 59 | 69 |
| FN（漏报） | 28 | 20 | 22 |

**解读**：

- **漏报（FN）**：RAG 最少（20），比 Baseline 减少了 28.6%。在合规监管场景中，漏报比误报代价更高，RAG 的低漏报率是重要优势。
- **误报（FP）**：Baseline 最少（51），Agent 最多（69）。Agent 倾向于将合规案例判定为违规。
- 三种方法均表现为"宁可误报、不愿漏报"的倾向（Recall > Precision），符合合规场景需求。

---

## 3. 质量层指标对比

| 指标 | Baseline | RAG | Agent |
|------|----------|-----|-------|
| **法律依据质量** | 0.785 | 0.757 | 0.699 |
| **推理质量** | 0.790 | 0.831 | 0.881 |

**解读**：

- **法律依据质量**：Baseline 最高（0.785）。RAG 引入法规检索后，模型引用的法规覆盖面更广但精准度略有下降。Agent 最低（0.699），可能因多节点流水线中法律引用被稀释。
- **推理质量**：Agent 最高（0.881），显著优于 Baseline（+11.5%）。Agent 的 5 步推理链（事实提取 → 数据校验 → 法律匹配 → 案例参考 → 结论）提供了更完整的推理过程。

---

## 4. 高级指标对比

| 指标 | Baseline | RAG | Agent |
|------|----------|-----|-------|
| **证据链完整性** | 0.000 | 0.311 | 0.589 |
| **法律引用准确性** | 0.000 | 0.151 | 0.160 |
| **整改建议可操作性** | 0.000 | 0.000 | 0.444 |
| **可解释性** | 0.030 | 0.260 | 0.337 |
| **结构化输出质量** | 0.040 | 0.305 | 0.320 |

**解读**：

高级指标体现了三种方法在"可用性"维度上的巨大差距：

1. **证据链完整性**：Agent（0.589）远超 RAG（0.311），Baseline 为零。Agent 的多步推理链为每个判定提供了可追溯的证据路径。
2. **整改建议可操作性**：仅 Agent 提供整改建议（0.444），这是 Agent 独有的 RemediationAdvisor 节点的价值体现。Baseline 和 RAG 无此能力。
3. **可解释性**：从 Baseline（0.030）→ RAG（0.260）→ Agent（0.337），逐级提升。法规检索和结构化推理均有助于提升输出的可解释性。
4. **法律引用准确性**：三种方法均较低（最高 0.160），说明模型在精确引用具体法条方面仍有较大改进空间。
5. **结构化输出质量**：RAG（0.305）和 Agent（0.320）接近，均远超 Baseline（0.040）。

## 5. 按违规类型细分分析

| 违规类型 | 数量 | Baseline | RAG | Agent |
|----------|------|----------|-----|-------|
| 合规（负例） | 280 | 81.7% | 78.9% | 75.4% |
| 不明码标价 | 228 | 95.6% | **97.4%** | 96.1% |
| 政府定价违规 | 120 | 92.5% | **95.0%** | 94.2% |
| 标价外加价 | 73 | 95.9% | 95.9% | 94.5% |
| 误导性价格标示 | 49 | 89.8% | 93.9% | **98.0%** |
| 未识别 | 15 | 93.3% | 86.7% | 93.3% |

（变相提高价格、虚假价格比较、虚假折扣等小类型三种方法均为 100%）

**关键发现**：

- **合规案例识别**是三种方法的共同弱点。Agent 的合规准确率最低（75.4%），即 24.6% 的合规案例被错误判定为违规。这与混淆矩阵中 Agent FP 最高（69）一致。
- **不明码标价**（最大违规子类）：RAG 最优（97.4%），法规检索增强在该类型上效果明显。
- **误导性价格标示**：Agent 表现最优（98.0%），多步推理有助于识别隐蔽的价格误导行为。
- **政府定价违规**：RAG 表现最优（95.0%），法规检索在匹配政府定价相关条款时更有针对性。

## 6. 性能与成本分析

| 指标 | Baseline | RAG | Agent |
|------|----------|-----|-------|
| 平均响应时间 | 5.85s | 6.00s | 25.6s |
| 相对成本 | 1x | 1.03x | 4.38x |

- RAG 仅比 Baseline 慢 2.6%，检索开销可忽略。
- Agent 是 Baseline 的 4.38 倍时延，主要来源于 5 节点串行调用（每个节点含 LLM 推理）。
- 在批量评测场景下，780 条数据 Agent 总耗时约 5.5 小时（vs Baseline 约 1.3 小时）。

## 7. 综合结论

### 7.1 三方法定位

| 方法 | 核心优势 | 核心劣势 | 适用场景 |
|------|----------|----------|----------|
| **Baseline** | 最佳精确率、最低时延、法律依据质量最高 | 无证据链、无可解释性、无整改建议 | 快速批量初筛 |
| **RAG** | 最高 F1 与 Recall、最少漏报、违规类型识别最均衡 | 合规识别略弱、法律依据质量下降 | 日常合规监测 |
| **Agent** | 最高推理质量、最高证据链完整性、唯一提供整改建议 | 时延最高、合规误报最多、准确率最低 | 深度案例分析 |

### 7.2 关键结论

1. **分类能力趋同**：三种方法在二分类准确率上差异不超过 1.54 个百分点，说明 Qwen-8B SFT 模型本身已具备较强的价格违规识别基础能力。
2. **RAG 是效果-成本最优解**：以极小的时延代价（+2.6%）换来最高的 F1（92.40%）和最低的漏报率，适合作为生产环境首选。
3. **Agent 的价值在可解释性而非准确率**：Agent 在证据链（0.589）、整改建议（0.444）、推理质量（0.881）上优势显著，适合需要完整审查链的深度分析场景。
4. **合规案例识别是共同瓶颈**：三种方法对合规案例的准确率均低于 82%，模型存在"过度判违"倾向，这是未来优化的重点方向。
5. **法律引用准确性普遍偏低**：三种方法的法律引用准确性均不超过 0.16，精确引用具体法条仍是技术难点。
