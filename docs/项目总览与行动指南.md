## 3. 下一步“最短路径”（建议你照着做）

### A. 先把微调跑起来（建议优先）

1. **确定基座模型**（二选一即可）
   - Qwen-7B（你 README 里写的）或 Qwen2.5-7B（中文/推理更强，常见）
2. **准备 LLaMA-Factory 训练配置**
   - 数据集：优先用 `train_chat_from_cases.jsonl` / `val_chat_from_cases.jsonl`
   - 方式：LoRA/QLoRA（更省显存）
3. **跑一个小实验闭环**
   - 先用较小的 epoch / steps（确保能收敛、能推理）
   - 保存权重到 `models/finetuned/`（建议新增该目录）

你完成 A 之后，就能在答辩里稳定讲清楚：**“我确实微调过，并且对比过。”**

### B. 再把知识库做成“能检索、能引用法条”

1. 将 `data/raw/laws/` 的法规文本做切分与向量化
2. Chroma 入库（先不做 Neo4j 也能交付）
3. 在 `HybridRetriever.retrieve()` 里先实现一个最简单的：
   - 关键词检索（法规标题/条号）
   - 向量检索（相似案例/相似法条）

### C. 最后把智能体节点从 TODO 变成可用（最小可用版本）

目标：在 `src/agents/coordinator.py` 里让流程真的产生可用输出。

- `IntentAnalyzer`: 先规则/正则抽取（平台、划线价、现价、时间窗口）
- `HybridRetriever`: Chroma top-k（法条 + 相似案例）
- `Grader`: 简单打分（包含条号/关键词更高分）
- `ReasoningEngine`: **先用基座模型/微调模型二选一**，按模板输出结构化结果
- `Reflector`: 先做一致性检查（是否引用法条、结论是否明确、是否给整改建议）
