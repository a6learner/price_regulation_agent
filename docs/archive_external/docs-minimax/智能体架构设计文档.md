## 📊 评估指标体系

### 1. 准确性指标 (Accuracy Metrics)
- **准确率 (Accuracy)**: 正确预测样本数 / 总样本数
- **精确率 (Precision)**: TP / (TP + FP)
- **召回率 (Recall)**: TP / (TP + FN)
- **F1分数**: 2 × (Precision × Recall) / (Precision + Recall)

### 2. 一致性指标 (Consistency Metrics)
- **Fleiss' Kappa**: 衡量与基座模型的一致性
- **Cohen's Kappa**: 衡量与人工标注的一致性
- **置信度分布**: 自熵指标的量化分析

### 3. 效率指标 (Efficiency Metrics)
- **响应时间**: 单次推理的平均耗时
- **吞吐量**: 每分钟处理的案例数
- **资源消耗**: GPU/CPU/内存使用率

### 4. 可解释性指标 (Interpretability Metrics)
- **推理链完整性**: CoT步骤的覆盖率
- **法规引用准确性**: 法律条文引用的正确性
- **证据链逻辑性**: 证据收集的逻辑连贯性

### 5. 实际业务指标 (Business Metrics)
- **违规检出率**: 相比传统方法的提升幅度
- **误报率**: 误判为违规的比例
- **漏报率**: 遗漏违规行为的比例

## 🔄 工作流程设计

### 典型案例处理流程
1. **输入**: "某酒店在携程划线价3000元，实际预订价198元"
2. **Intent分析**: 识别为"价格欺诈检测"任务
3. **混合检索**: 找到相关法条和类似案例
4. **质量评估**: 筛选最相关的法律依据
5. **推理分析**: 使用微调模型进行CoT推理
6. **反思确认**: 二次检查逻辑和法规适用
7. **输出**: 结构化的违规判定报告

### 批量监测流程
1. **数据接入**: 从电商平台获取价格数据
2. **异常检测**: 初步筛选可能违规的案例
3. **智能体分析**: 对疑似案例进行深度分析
4. **分级预警**: 根据违规严重程度分级
5. **证据生成**: 自动生成证据包
6. **人工审核**: 关键案例人工确认
7. **反馈学习**: 根据审核结果优化模型

## 🛠️ 技术实现要点

### 1. 数据处理Pipeline
- **数据采集**: 支持多平台API和爬虫
- **数据清洗**: 文本预处理、实体识别、去噪
- **数据标注**: 半自动标注工具
- **质量控制**: 数据一致性检查

### 2. 模型微调策略
- **基础模型**: Qwen-7B
- **微调方法**: LoRA + QLoRA (4-bit量化)
- **训练数据**: CoT格式的200-500条样本
- **验证策略**: 1500-2500条真实商品数据验证

### 3. 系统部署架构
- **API服务**: FastAPI + uvicorn
- **数据库**: Chroma + Neo4j
- **消息队列**: Redis (可选)
- **监控**: Prometheus + Grafana
