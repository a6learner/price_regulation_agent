# 面向价格回归监督的智能体 - 架构设计

## 五层智能体架构

```
┌────────────────────────────────────────────────────────────────────────────────┐
│                           用户输入层                                   │
│                    价格监测任务 / 用户查询 / 投诉案例                      │
└────────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌────────────────────────────────────────────────────────────────────────────────┐
│                      智能体协调层 (LangGraph)                          │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐     │
│  │ Intent Analyzer  │───▶│ Hybrid Retriever│───▶│     Grader     │     │
│  │   意图分析     │    │   混合检索      │    │   质量评估     │     │
│  └─────────────────┘    └─────────────────┘    └─────────────────┘     │
│        │                     │                     │                     │
│        ▼                     ▼                     ▼                     │
│  ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐     │

│  │  违规判定报告   │                  │ Neo4j 知识图谱  │                │
│  └─────────────────┘                  └─────────────────┘                │
└────────────────────────────────────────────────────────────────────────────────┘
```

## 各层详细说明

### 1. 感知层 / 用户输入层
**功能**: 接收价格监测任务和用户查询

**输入类型**:
- 批量商品价格数据
- 单一价格违规投诉
- 历史价格对比请求

### 2. 智能体协调层 (核心)

#### 2.1 Intent Analyzer - 意图分析节点
**功能**: 识别查询意图和任务类型

**输出**:
```json
{
  "task_type": "price_fraud_detection",
  "priority": "high",
  "violation_type": "虚构原价",
  "keywords": ["划线价", "成交价", "虚标"]
}
```

#### 2.2 Hybrid Retriever - 混合检索节点
**功能**: 同时调用向量数据库和知识图谱

**检索策略**:
- 向量检索: 基于语义相似度查找相关案例
- 图谱检索: 查找法律条款关系和案例关联
- 关键词检索: 精确匹配法律条文号

#### 2.3 Grader - 质量评估节点
**功能**: 评估检索结果的相关性和可信度

**评估维度**:
- 内容相关性 (0-1分)
- 时效性 (最新法规优先)
- 权威性 (国家级 > 地方级)
- 完整性

#### 2.4 Reasoning Engine - 推理引擎
**功能**: 基于微调的Qwen-7B进行CoT推理

**推理过程** (Chain of Thought):
```
Step 1: 引用相关法条
Step 2: 事实数据比对
Step 3: 计算溢价率/折扣率
Step 4: 逻辑一致性验证
Step 5: 得出违规判定结论
```

#### 2.5 Reflector - 自反思节点
**功能**: 二次检查结论，确保准确性

**检查项**:
- 法规是否为最新版本
- 引用的法条号是否正确
- 逻辑推导是否自洽
- 结论是否支持证据

### 3. 知识库层

#### 3.1 Chroma 向量数据库
**存储内容**:
- 法律条文全文
- 处罚案例描述
- 违规模式向量

#### 3.2 Neo4j 知识图谱
**图结构**:
```
(法律) --[包含条款]--> (条款)
(条款) --[禁止行为]--> (违规类型)
(违规类型) --[对应案例]--> (案例)
(案例) --[违反]--> (条款)
(案例) --[涉及企业]--> (企业)
```

## 技术栈

| 层次 | 技术选型 | 说明 |
|-----|---------|------|
| 协调层 | LangGraph | 智能体工作流编排 |
| 推理模型 | Qwen-7B + LLaMA-Factory | 本地微调，可控可解释 |
| 向量库 | ChromaDB | 语义相似度检索 |
| 知识图谱 | Neo4j | 法律条款关系管理 |
| API服务 | FastAPI | RESTful接口 |
| 数据处理 | Pandas + NumPy | 数据清洗和特征提取 |

## 核心创新点

1. **混合检索架构**: 向量+图谱+关键词，提高检索准确性
2. **CoT推理**: 可解释的推理过程，符合法律逻辑
3. **自反思机制**: 二次验证，降低误判率
4. **本地微调**: 基于行政处罚文书的领域适应

## 系统工作流程

```
输入: "某酒店划线价3000元，实际198元，无前7日成交"
  │
  ├─→ IntentAnalyzer: 识别为虚构原价类型
  │
  ├─→ HybridRetriever: 检索《明码标价规定》第16条 + 相似案例
  │
  ├─→ Grader: 筛选出最高相关度的法条和案例
  │
  ├─→ ReasoningEngine:
  │      ├─ 引用第16条: 被比较价应不高于最后一次交易价
  │      ├─ 数据对比: 3000元 vs 198元，差额15倍
  │      ├─ 结论: 划线价不真实
  │
  ├─→ Reflector: 二次确认法规和逻辑无误
  │
  └─→ 输出: "违规。构成价格欺诈，依据第16条"
```
