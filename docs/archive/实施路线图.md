# 项目实施路线图

## 🎯 项目目标

构建一个智能体系统，输入商品价格信息，输出是否违规及依据。

## 📅 推荐实施顺序

### ✅ 第1步：提取PDF结构化信息（当前任务）

**为什么先做这一步？**
- 这是所有后续工作的基础
- 133份PDF包含了丰富的违规案例和法律推理逻辑
- 提取的结构化数据将用于：
  - 构建训练集（CoT格式）
  - 构建知识库（案例库）
  - 作为评估的Ground Truth

**实施步骤**:
1. 运行PDF提取工具
2. 检查提取质量
3. 人工修正错误
4. 保存结构化数据

**预计时间**: 1-2天

**命令**:
```bash
cd price_regulation_agent/src/data_collection
python pdf_extractor.py
```

---

### ✅ 第2步：构建CoT训练集

**为什么第二步？**
- 需要将提取的结构化数据转换为模型可训练的格式
- CoT格式是微调模型的关键

**实施步骤**:
1. 读取提取的结构化数据
2. 转换为CoT格式（input + thought + output）
3. 数据清洗和去重
4. 保存为JSONL格式

**预计时间**: 1-2天

**输出**: `data/training/sft_samples_expanded.jsonl` (200-500条)

---

### ✅ 第3步：构建知识库

**为什么第三步？**
- 知识库是智能体检索法律条文和案例的基础
- 向量库用于语义检索
- 知识图谱用于法律条款关系

**实施步骤**:
1. 读取法律条文（data/raw/laws/）
2. 将法律条文存入Chroma向量库
3. 将案例存入向量库
4. （可选）构建Neo4j知识图谱

**预计时间**: 2-3天

**工具**: ChromaDB, Neo4j

---

### ✅ 第4步：采集商品数据（并行进行）

**为什么可以并行？**
- 商品数据主要用于评估，不依赖训练
- 可以在模型训练的同时进行采集

**实施步骤**:
1. 选择采集方法（推荐：慢慢买/EasySpider）
2. 批量采集1500-2500条商品数据
3. 数据清洗和去重
4. 保存为JSONL格式

**预计时间**: 1-2周（取决于采集方法）

**输出**: `data/validation/product_data.jsonl`

---

### ✅ 第5步：模型微调

**为什么第五步？**
- 需要先有训练集和知识库
- 微调是提升模型性能的关键

**实施步骤**:
1. 准备微调环境（AutoDL/本地GPU）
2. 使用LLaMA-Factory加载Qwen-7B
3. 使用CoT训练集进行LoRA微调
4. 验证微调效果

**预计时间**: 3-5天（取决于GPU）

**工具**: LLaMA-Factory, QLoRA

---

### ✅ 第6步：实现智能体架构

**为什么最后一步？**
- 需要先有微调模型和知识库
- 智能体是系统的核心，整合所有组件

**实施步骤**:
1. 实现Intent Analyzer
2. 实现Hybrid Retriever
3. 实现Grader
4. 实现Reasoning Engine（调用微调模型）
5. 实现Reflector
6. 使用LangGraph编排工作流

**预计时间**: 1-2周

**工具**: LangGraph, FastAPI

---

### ✅ 第7步：评估和优化

**最后一步**:
1. 使用商品数据评估智能体性能
2. 对比基座模型和微调模型
3. 分析错误案例，优化模型
4. 生成评估报告

**预计时间**: 1周

---

## 💡 为什么推荐这个顺序？

1. **数据优先**: 先有数据，才能训练和评估
2. **依赖关系**: 后面的步骤依赖前面的结果
3. **并行优化**: 商品数据采集可以与其他步骤并行
4. **迭代改进**: 每个步骤完成后可以立即验证效果

## ⚠️ 注意事项

1. **数据质量**: PDF提取可能不完美，需要人工检查
2. **资源需求**: 模型微调需要GPU（建议AutoDL）
3. **时间管理**: 商品数据采集可能需要较长时间
4. **版本控制**: 建议使用Git管理代码和数据
