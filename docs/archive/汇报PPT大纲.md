## 第2页: 研究背景与问题

### 研究背景
- 电商行业快速发展，价格行为日益复杂
- 传统监管模式（人工抽检）效率低下
- 价格违规手段不断翻新（大数据杀熟、虚构原价等）

### 核心问题
1. 如何实现全天候、自动化的价格监测？
2. 如何准确识别复杂的违规行为模式？
3. 如何保证监管决策的可解释性和合规性？

### 研究意义
- 提升监管效率，降低人工成本
- 保护消费者权益，维护市场公平
- 推动监管科技(RegTech)发展

## 第4页: 五层智能体架构 (核心页面2)

### 智能体工作流
```
输入查询
    │
    ▼
┌─────────────────────────────────────────────┐
│ Intent Analyzer → 意图分析               │
│ Hybrid Retriever → 混合检索(向量+图谱)   │
│ Grader → 质量评估                     │
│ Reasoning Engine → CoT推理(微调模型)      │
│ Reflector → 自反思验证                  │
└─────────────────────────────────────────────┘
    │
    ▼
结构化输出: 违规判定报告
```

### 核心特点
- **混合检索**: 向量相似度 + 知识图谱关系 + 关键词精确匹配
- **CoT推理**: 可解释的逐步推理过程
- **自反思机制**: 二次验证，降低误判

## 可能的问题准备

### Q1: 为什么选择本地微调而不是直接调用API？
A:
- 成本可控：无需持续API调用费用
- 数据隐私：行政处罚文书涉及敏感信息
- 可定制性：深度适配法律推理需求
- 学术价值：微调过程本身就是毕设的核心贡献

### Q2: 微调数据从哪里来？
A:
- 主要来源：信用中国、地方市场监管局官网的行政处罚决定书
- 数据格式：从非结构化文书转换为CoT格式的指令样本
- 预计规模：200-500条高质量样本

### Q3: 预计能达到什么效果？
A:
- 目标准确率：85%以上
- 相比基座模型的提升：通过CoT推理和领域知识，在法律适用性和逻辑性上有明显提升
- 评估方式：LLM-as-a-Judge + 人工抽样验证

### Q4: 如果微调效果不好怎么办？
A:
- 备选方案：基座模型Agent + 强化Prompt工程
- 分层处理：简单规则用引擎，复杂推理用LLM
- 持续迭代：根据评估结果调整数据和参数
