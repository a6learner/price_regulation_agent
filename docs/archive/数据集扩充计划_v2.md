## 目标

## 核心关注指标

扩充后的数据集需要确保以下指标的质量和覆盖：

| 指标 | 重要性 | 说明 |
|------|--------|------|
| **is_violation** | ⭐⭐⭐ | 是否违规的二分类判断 |
| **violation_type** | ⭐⭐⭐ | 违规类型的准确分类（5类） |
| **reasoning_chain** | ⭐⭐⭐ | 推理链完整性（事实→分析→结论） |
| **legal_basis** | ⭐⭐ | 法律依据的准确性 |
| **complexity** | ⭐⭐ | 案例复杂度分布（simple/medium/complex） |
| platform | ⭐ | 平台分布（不强制均衡） |

### Step 3: 创建合规案例生成脚本
- **状态**: ✅ 已完成
- **脚本**: `scripts/generate_compliance_variants.py` (新建)
- **功能**:
  1. 读取违规案例
  2. 通过LLM生成"合规版本"（修改关键事实使其不构成违规）
  3. 确保推理链完整（事实要点 → 合规分析 → 结论）
  4. 输出eval格式

- **生成策略（按违规类型）**:
  | 违规类型 | 合规变体逻辑 |
  |---------|-------------|
  | 虚构原价 | 有真实成交记录的原价标注 |
  | 虚假折扣 | 折扣与实际优惠一致 |
  | 价格误导 | 价格标注清晰准确 |
  | 要素缺失 | 完整标注所有必要信息 |
  | 其他 | 符合价格法规要求 |

- **完成时间**: 2026-04-05
- **备注**: 脚本创建成功，集成MaaS API调用
