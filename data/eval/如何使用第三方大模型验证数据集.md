# 如何使用第三方大模型验证数据集

## 概述

本指南说明如何使用第三方大语言模型（如Gemini、GPT-4、Claude等）来验证和改进我们的评估数据集。

---

## 准备材料

你需要准备以下三个文件发送给第三方大模型：

### 1. 📋 数据集质量评估指南.md
**作用**: 详细说明数据集的质量标准、格式规范、评估维度
**位置**: `data/eval/数据集质量评估指南.md`
**内容**: 8个章节，包括格式规范、质量标准、常见问题、补充建议等

### 2. 📝 数据集验证Prompt.md
**作用**: 具体的验证任务说明和输出格式要求
**位置**: `data/eval/数据集验证Prompt.md`
**内容**: 明确的任务描述、评估要求、输出模板

### 3. 📊 eval_100.jsonl（数据集本身）
**作用**: 待验证的评估数据集
**位置**: `data/eval/eval_100.jsonl`
**内容**: 104条评估案例（JSONL格式）

### 4. 📈 eval_meta.json（可选）
**作用**: 数据集的统计信息
**位置**: `data/eval/eval_meta.json`
**内容**: 分布统计、平台分布、违规类型分布等

---

## 使用步骤

### 步骤1: 准备材料

```bash
cd price_regulation_agent/data/eval

# 确认文件存在
ls -la 数据集质量评估指南.md
ls -la 数据集验证Prompt.md
ls -la eval_100.jsonl
ls -la eval_meta.json
```

### 步骤2: 选择验证平台

推荐使用以下大模型之一：

| 模型 | 优势 | 推荐场景 |
|------|------|---------|
| **Gemini 2.0 Flash** | 免费、快速、上下文窗口大 | 首选，性价比最高 |
| **GPT-4o** | 准确性高、法律推理强 | 最终验证，需要最高质量 |
| **Claude 3.5 Sonnet** | 结构化输出好 | 需要严格格式化输出 |
| **Qwen-Max** | 中文理解好 | 国内平台验证 |

### 步骤3: 发送验证请求

#### 方案A: 分批验证（推荐，避免超token限制）

**第一轮：阅读指南**
```
请仔细阅读以下两个文档，了解数据集验证任务：

[粘贴 数据集质量评估指南.md 的完整内容]

---

[粘贴 数据集验证Prompt.md 的完整内容]

请确认你已理解验证任务的要求。如果理解了，请回复"已理解验证任务要求"。
```

**第二轮：提供数据集统计**
```
以下是数据集的统计信息：

[粘贴 eval_meta.json 的内容]

请先根据这些统计信息，初步评估数据分布是否合理。
```

**第三轮：验证数据集（分批）**
```
现在开始验证数据集。我将分批提供数据，每批约20-30条。

【第1批：eval_001 到 eval_030】

[粘贴前30条JSONL数据]

请按照验证Prompt中的格式，输出这批数据的验证结果。
```

重复第三轮，直到所有数据验证完成。

**第四轮：生成总体报告**
```
你已经完成了所有104条数据的验证。现在请：

1. 汇总所有问题
2. 统计整体质量评分
3. 提供补充数据建议
4. 生成3-5个高质量的补充案例样本
5. 输出完整的验证报告（按照Prompt中的模板）
```

#### 方案B: 一次性验证（适用于支持大上下文的模型）

```
# 任务说明
你是一个数据集质量验证专家。请帮我验证一个电商价格合规评估数据集。

# 评估指南
[粘贴 数据集质量评估指南.md 的完整内容]

# 验证任务要求
[粘贴 数据集验证Prompt.md 的完整内容]

# 数据集统计
[粘贴 eval_meta.json 的内容]

# 待验证数据集（104条）
[粘贴 eval_100.jsonl 的完整内容]

---

请严格按照验证Prompt中的格式，输出完整的验证报告。
```

### 步骤4: 收集验证结果

将大模型返回的验证报告保存为：
```
data/eval/验证报告_[模型名称]_[日期].md
```

例如：
- `验证报告_Gemini2.0_20260311.md`
- `验证报告_GPT4o_20260311.md`

### 步骤5: 根据报告改进数据集

根据验证报告中的建议：

1. **修正严重问题**
   ```bash
   # 修正格式错误、一致性问题等
   python scripts/fix_eval_issues.py --report 验证报告_xxx.md
   ```

2. **补充缺失数据**
   ```bash
   # 将大模型生成的补充数据添加到数据集
   # 手动审核后追加到 eval_100.jsonl
   ```

3. **重新统计**
   ```bash
   # 更新统计信息
   python scripts/数据处理/generate_eval_meta.py
   ```

### 步骤6: 交叉验证（可选）

使用不同的大模型重复步骤2-5，对比验证结果：

```bash
# 对比两个验证报告
diff 验证报告_Gemini2.0_20260311.md 验证报告_GPT4o_20260311.md
```

## 实际操作示例

### 示例1: 使用Gemini 2.0 Flash验证（免费）

1. 访问 Google AI Studio (https://aistudio.google.com/)
2. 创建新对话
3. 设置系统指令：
   ```
   你是一个专业的数据集质量验证专家，擅长评估数据质量、识别问题并提供改进建议。
   ```
4. 按照"方案A"分批发送内容
5. 收集并整理验证报告

### 示例2: 使用ChatGPT Plus（GPT-4o）验证

1. 访问 ChatGPT (https://chat.openai.com/)
2. 上传文件：
   - `数据集质量评估指南.md`
   - `数据集验证Prompt.md`
   - `eval_100.jsonl`
3. 发送指令：
   ```
   请阅读上传的三个文件，按照验证Prompt的要求，完成数据集质量验证任务。
   ```
4. 下载验证报告

### 示例3: 使用API批量验证（编程方式）

```python
import anthropic  # 或 openai, google.generativeai

# 读取文件
with open('数据集质量评估指南.md', 'r', encoding='utf-8') as f:
    guide = f.read()

with open('数据集验证Prompt.md', 'r', encoding='utf-8') as f:
    prompt = f.read()

with open('eval_100.jsonl', 'r', encoding='utf-8') as f:
    dataset = f.read()

# 构建完整prompt
full_prompt = f"""
{guide}

---

{prompt}

---

待验证数据集：
{dataset}
"""

# 调用API
client = anthropic.Anthropic(api_key="your_api_key")
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=16000,
    messages=[
        {"role": "user", "content": full_prompt}
    ]
)

# 保存结果
with open('验证报告_Claude3.5_20260311.md', 'w', encoding='utf-8') as f:
    f.write(response.content[0].text)
```

## 补充数据集的流程

根据验证报告补充数据：

### 方法1: 让大模型生成（快速）

**Prompt模板**:
```
根据验证报告，我需要补充以下类型的案例：

1. 要素缺失类案例 5条（未标明促销时间、运费未明示等）
2. 简单难度合规案例 10条
3. 服务类场景案例 5条（酒店、餐饮、医疗等）

请严格按照以下格式为每条案例生成完整的JSONL数据：

[粘贴一个高质量案例作为示例]

要求：
- 案例真实、具体、可判断
- 标准答案结构完整（事实要点+合规分析+结论+整改建议）
- 法条引用准确
- meta信息与内容一致

请开始生成。
```

### 方法2: 基于真实案例改编（高质量）

```bash
# 从133条真实处罚案例中选择合适的案例
python scripts/数据处理/select_cases_for_eval.py \
  --source data/sft/processed/extracted_cases.jsonl \
  --target_type "要素缺失" \
  --count 5

# 人工审核并改编为评估格式
```

### 方法3: 混合方式（推荐）

1. 让大模型快速生成初稿（50条）
2. 人工审核，保留高质量的（30条）
3. 基于真实案例改编补充（20条）
4. 再次验证新数据

## 常见问题

### Q1: Gemini说"上下文过长"怎么办？
**A**: 使用"方案A"分批验证，每批20-30条数据。

### Q2: 验证报告不完整，缺少某些部分？
**A**: 明确提醒大模型："请严格按照验证Prompt中的输出模板，输出完整的验证报告，包括9个部分"。

### Q3: 大模型生成的补充案例质量不高？
**A**:
- 提供更多高质量示例
- 明确指出常见错误
- 要求逐条生成并人工审核

### Q4: 不同大模型的验证结果差异很大？
**A**: 这是正常的，建议：
- 识别共同指出的问题（这些问题更可靠）
- 对争议案例进行人工复核
- 以法律准确性为最高优先级

### Q5: 如何判断数据集质量是否足够？
**A**: 参考以下标准：
- 整体评分 ≥ 85分：优秀，可直接使用
- 整体评分 70-84分：良好，小幅修正后使用
- 整体评分 60-69分：合格，需要一定修正
- 整体评分 < 60分：不合格，需要大幅完善
