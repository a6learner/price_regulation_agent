# 电商价格合规评估数据集 - 质量评估指南

## 文档目的

本文档用于指导第三方验证者（包括人类审核员和大语言模型）评估本评估数据集的质量和有效性。数据集将用于对比评估三种价格合规分析系统的性能：
1. **Baseline**: 纯LLM API调用
2. **RAG系统**: LLM + 检索增强（法条+案例）
3. **Agent系统**: LLM + RAG + 多步推理

## 2. 数据格式规范

### 2.1 JSONL格式要求

每条数据必须是一个独立的JSON对象，包含两个顶层字段：`messages` 和 `meta`。

**完整示例**:
```json
{
  "messages": [
    {
      "role": "system",
      "content": "你是一名电商平台价格合规审查助手，熟悉《价格法》《明码标价和禁止价格欺诈规定》及相关配套规章。你需要根据给定的案件事实，做出法律分析并给出是否违规的结论和依据。"
    },
    {
      "role": "user",
      "content": "某监管部门查处了一起电商价格违法案件。\n该经营者通过淘宝平台开展经营活动。\n案情概述：[具体案情描述]\n与价格相关的关键事实包括：[关键事实列举]\n监管机关认定的违法类型为：[违法类型或\"无违法\"]\n请根据上述事实，从价格合规的角度分析该经营行为是否违规，并给出依据和结论。"
    },
    {
      "role": "assistant",
      "content": "事实要点：\n- [提取关键事实]\n\n合规分析：\n- [法规条文分析]\n- [行为定性分析]\n\n结论：[违规/不违规]。[详细结论说明]\n\n整改建议：\n- [建议1]\n- [建议2]"
    }
  ],
  "meta": {
    "case_id": "eval_001",
    "is_violation": true,
    "violation_type": "虚构原价",
    "platform": "淘宝",
    "scenario": "限时折扣",
    "complexity": "simple"
  }
}
```

### 2.2 字段说明

#### messages字段（必需）
OpenAI Chat Completion格式的对话列表。

**system消息**:
- 固定角色设定
- 说明助手的职责和能力范围
- 强调需要依据法律法规做出判断

**user消息**:
- 案情描述的结构化呈现
- 必须包含：平台信息、案情概述、关键事实、监管机关认定（或"请分析"）
- 真实、具体、可判断

**assistant消息**（标准答案）:
- **事实要点**: 从案情中提取的关键信息（3-8条）
- **合规分析**: 法规适用分析和行为定性（2-5段）
- **结论**: 明确的违规/不违规判定 + 法条引用
- **整改建议**: 具体可操作的改进措施（1-3条）

#### meta字段（必需）
记录案例的元信息，用于统计和分层评估。

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| case_id | string | 唯一案例ID | eval_001, eval_002, ... |
| is_violation | boolean | 是否违规 | true / false |
| violation_type | string | 违规类型 | 虚构原价/虚假折扣/价格误导/要素缺失/其他/无 |
| platform | string | 电商平台 | 淘宝/京东/拼多多/美团/抖音/天猫/其他 |
| scenario | string | 业务场景 | 限时折扣/大促活动/直播带货/... |
| complexity | string | 复杂度 | simple / medium / complex |

## 3. 数据质量评估标准

### 3.1 核心质量标准（必须满足）

#### 标准1: 真实性与合理性
- [ ] 案情描述符合真实电商场景
- [ ] 价格数值在合理范围内（不过分夸张）
- [ ] 违规行为与现实中常见案例一致
- [ ] 平台特征与实际平台规则相符

**反例**:
- ❌ "某商品原价99999元，现价9.9元"（折扣过分夸张）
- ❌ "拼多多平台使用京东物流发货"（平台特征不符）
- ✅ "某商品原价899元从未成交，实际促销价299元"（真实合理）

#### 标准2: 可判定性
- [ ] 案情描述包含足够的判断依据
- [ ] 关键事实明确、不模糊
- [ ] 违规点清晰可识别（或明确不违规）
- [ ] 不存在信息缺失导致无法判断的情况

**反例**:
- ❌ "某商品价格有问题"（信息不足，无法判断）
- ❌ "商品标注原价XX元"（未说明是否真实成交）
- ✅ "商品标注原价899元，经查从未以此价格成交"（信息充分）

#### 标准3: 标准答案质量
- [ ] **事实要点**: 准确提取案情中的关键信息，3-8条，格式统一
- [ ] **合规分析**: 引用相关法律法规，逻辑清晰，2-5段
- [ ] **结论**: 明确判定违规/不违规，并引用具体法条
- [ ] **整改建议**: 具体可操作，1-3条

**assistant回复必备要素**:
1. 结构完整（事实要点 + 合规分析 + 结论 + 整改建议）
2. 法条引用准确（《价格法》《明码标价和禁止价格欺诈规定》等）
3. 逻辑严密（分析→结论→建议 环环相扣）
4. 语言专业（符合法律文书规范）

#### 标准4: 法律准确性
- [ ] 违规类型定性准确
- [ ] 引用的法条存在且相关
- [ ] 法律推理逻辑正确
- [ ] 整改建议符合法律要求

**常见违规类型定义**:
- **虚构原价**: 标注的"原价/划线价"从未实际成交，或无交易依据
- **虚假折扣**: 声称"折扣/优惠"但价格未实际下降，或优惠条件虚假
- **价格误导**: 价格标示方式使消费者产生误解（如隐藏条件、不实宣传）
- **要素缺失**: 未明示促销时间/数量/条件等关键价格信息

**主要法律依据**:
- 《中华人民共和国价格法》第十四条
- 《明码标价和禁止价格欺诈规定》第六条、第十七条、第十九条
- 《价格违法行为行政处罚规定》第七条

## 4. 常见质量问题及修正建议

### 4.1 案情描述问题

#### 问题1: 信息不足，无法判断
**示例**:
> "某商品在促销期间价格有问题"

**问题**: 未说明具体价格、原价、促销价、成交记录等

**修正**:
> "某商品标注原价899元，促销价299元，经查该商品从未以899元成交，过去30天日常售价为280-320元"

#### 问题2: 信息矛盾
**示例**:
> "商品原价100元，促销价200元，优惠力度很大"

**问题**: 促销价高于原价，逻辑矛盾

**修正**: 核实数值，确保逻辑一致

#### 问题3: 场景不真实
**示例**:
> "某商品原价99999元，现价1元"

**问题**: 价格差异过大，不符合真实场景

**修正**: 使用合理的价格区间

### 4.2 标准答案问题

#### 问题1: 结构不完整
**示例**:
> "该行为违规，依据《价格法》"

**问题**: 缺少事实要点、合规分析、整改建议

**修正**: 按照标准格式补全所有部分

#### 问题2: 法条引用错误
**示例**:
> "依据《消费者权益保护法》第99条"

**问题**: 引用了不存在的条款

**修正**: 使用正确的法律条文

#### 问题3: 结论与meta不一致
**示例**:
- assistant结论: "不违规"
- meta.is_violation: true

**问题**: 标准答案与元信息矛盾

**修正**: 确保一致性

### 4.3 元信息问题

#### 问题1: violation_type与结论不匹配
**示例**:
- meta.violation_type: "虚构原价"
- 实际违规行为: 虚假折扣

**修正**: 重新审查案情，正确分类

#### 问题2: complexity标注不准确
**simple**: 特征明显，一眼可判断（如"从未成交的原价"）
**medium**: 需要推理，涉及多个条件（如"优惠条件复杂"）
**complex**: 边界case，涉及法律灰色地带

- [ ] 案情描述真实、具体、可判断
- [ ] 关键事实明确列出
- [ ] 平台和场景符合实际
- [ ] 价格数值合理

- [ ] 事实要点完整（3-8条）
- [ ] 合规分析有法条引用
- [ ] 结论明确（违规/不违规）

**一致性检查**:
- [ ] assistant结论与meta.is_violation一致

### 5.2 数据集整体检查清单

**分布检查**:
- [ ] 违规/合规比例在55%-65% / 35%-45%之间

- [ ] 场景覆盖充分（至少10种不同场景）

**区分度检查**:
- [ ] 包含明显违规案例（easy cases）
- [ ] 包含需要推理的案例（medium cases）
- [ ] 包含边界案例（hard cases）
- [ ] 合规案例有足够区分度（非简单的"完全符合规定"）

## 7. 使用场景说明

### 7.1 系统评估场景

本数据集将用于评估三种系统在价格合规判定任务上的性能：

#### Baseline（纯LLM）
- 输入: user消息内容
- 输出: LLM生成的分析结果
- 对比: 与标准答案（assistant）对比

#### RAG系统
- 输入: user消息 + 检索到的法条/案例
- 输出: RAG增强的分析结果
- 对比: 与标准答案对比

#### Agent系统
- 输入: user消息
- 过程: 多步推理（意图分析→检索→评分→推理→反思）
- 输出: Agent的最终结果
- 对比: 与标准答案对比

### 7.2 评估指标

**自动评估指标**:
- 准确率: (TP + TN) / Total
- 精确率: TP / (TP + FP)
- 召回率: TP / (TP + FN)
- F1-Score
- 违规类型识别准确率

**人工评估指标**:
- 法条引用准确性
- 分析逻辑完整性
- 整改建议实用性

**分层评估**:
- 按复杂度分层: simple / medium / complex
- 按违规类型分层: 虚构原价 / 虚假折扣 / 价格误导 / 要素缺失
- 按平台分层: 淘宝 / 京东 / 拼多多 / 美团 / 抖音等

## 附录: 法律依据速查

### 常用法条

**《中华人民共和国价格法》**:
- 第十四条: 经营者不得利用虚假的或者使人误解的价格手段，诱骗消费者或者其他经营者与其进行交易。

**《明码标价和禁止价格欺诈规定》**:
- 第六条: 经营者不得在标价之外加价出售商品或者提供服务，不得收取任何未予标明的费用。
- 第十七条: 经营者不得实施下列价格欺诈行为...（含虚构原价、虚假优惠等）
- 第十九条: 经营者折价、减价，应当标明或者通过其他方便消费者认知的方式表明折价、减价的基准。

**《价格违法行为行政处罚规定》**:
- 第七条: 经营者违反价格法规定，利用虚假的或者使人误解的价格手段，诱骗消费者或者其他经营者与其进行交易的，责令改正，没收违法所得，并处违法所得5倍以下的罚款...
