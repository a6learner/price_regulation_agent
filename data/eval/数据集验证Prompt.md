# 数据集质量验证任务 - Prompt

## 任务概述

你将作为第三方验证者，评估一个用于**电商价格合规智能体**性能评估的数据集质量。该数据集将用于对比三种系统（纯LLM、RAG系统、Agent系统）的性能，因此数据质量直接影响实验结论的可靠性。

**数据集基本信息**:
- 名称: 电商价格合规评估数据集 v1.0
- 格式: JSONL
- 当前规模: 104条
- 用途: 评估价格合规判定系统的准确性、法条引用能力、推理能力

---

## 你的任务

### 任务1: 全面质量评估（必做）
检查数据集的整体质量，包括：
1. **格式规范性**: JSON格式、字段完整性
2. **内容质量**: 真实性、可判定性、多样性
3. **标准答案质量**: 结构完整性、法律准确性
4. **一致性**: meta信息与内容的一致性
5. **分布合理性**: 违规类型、平台、复杂度的分布

### 任务2: 问题识别与修正建议（必做）
- 识别每条数据的问题（如有）
- 按严重程度分类：严重/中等/轻微
- 提供具体的修正建议
- 标注需要删除或重写的案例

### 任务3: 补充数据建议（必做）
基于分布分析和覆盖度评估，提供：
- 需要补充的数据类型和数量
- 建议的案例方向（平台、场景、违规类型）
- 3-5个高质量的补充案例样本

### 任务4: 生成补充数据（可选）
如果你有能力，请直接生成10-20条高质量的补充数据，填补当前数据集的不足。

## 评估标准（详见配套文档）

请参考《数据集质量评估指南.md》中的详细标准，重点关注：

### 核心质量标准（第3节）
1. **真实性与合理性**: 案情符合真实电商场景，价格合理，平台特征准确
2. **可判定性**: 信息充分，关键事实明确，违规点清晰
3. **标准答案质量**: 结构完整（事实要点+合规分析+结论+整改建议），法条准确
4. **法律准确性**: 违规类型定性正确，法条引用准确，推理逻辑正确

### 分布目标（第3.2节）
- 违规/合规: 60% / 40%
- 虚构原价: 18%, 虚假折扣: 18%, 价格误导: 15%, 要素缺失: 7%
- simple: 60%, medium: 30%, complex: 10%
- 平台分布均匀（7个主流平台）

### 多样性要求（第3.3节）
- 场景覆盖: 至少10种不同业务场景
- 商品多样: 实物/服务/虚拟商品
- 平台特色: 体现各平台特有的促销机制

### 如何检查一致性
```python
def check_consistency(item):
    conclusion = item["messages"][2]["content"]
    is_violation = item["meta"]["is_violation"]

    if "违规" in conclusion and not is_violation:
        return "不一致"
    if "不违规" in conclusion and is_violation:
        return "不一致"
    return "一致"
```
